# PavoPDF — Plan 7: Distribution & Website

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship PavoPDF v0.1.0 to the world — finalize the app bundle config, generate platform icons, wire up the Tauri auto-updater, configure GitHub Actions to build signed releases for all three platforms on every `v*` tag, and publish the pavopdf.com SvelteKit landing page.

**Architecture:** The Tauri bundler produces platform-native installers (NSIS on Windows, DMG on macOS, AppImage + DEB on Linux) that are uploaded as draft GitHub Releases by the CI workflow. Auto-updates are opt-in, driven by Tauri's built-in updater pointing at the GitHub Releases JSON endpoint. The marketing website is a static SvelteKit app deployed to Cloudflare Pages, sharing the same color palette as the desktop app.

**Tech Stack:** Tauri bundler, GitHub Actions, SvelteKit 2, Tailwind CSS 3, Cloudflare Pages

**Depends on:** Plans 1–6 complete

### Task 3: macOS Entitlements

**Files:**
- Create: `src-tauri/entitlements.plist`

- [ ] **Step 1: Write the entitlements file**

Create `src-tauri/entitlements.plist`:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <!-- Allow the app to run without being sandboxed (required for file system access to arbitrary user-chosen PDFs) -->
    <key>com.apple.security.app-sandbox</key>
    <false/>

    <!-- Hardened runtime: allow JIT compilation (needed by WebKit / WKWebView) -->
    <key>com.apple.security.cs.allow-jit</key>
    <true/>

    <!-- Hardened runtime: allow unsigned executable memory (needed by some WebKit internals) -->
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>

    <!-- Hardened runtime: disable library validation so third-party dynamic libraries load -->
    <key>com.apple.security.cs.disable-library-validation</key>
    <true/>

    <!-- Read/write access to user-selected files via NSOpenPanel / NSSavePanel -->
    <key>com.apple.security.files.user-selected.read-write</key>
    <true/>

    <!-- Read access to the Downloads folder (common source of PDFs) -->
    <key>com.apple.security.files.downloads.read-write</key>
    <true/>
</dict>
</plist>
```

**Why `app-sandbox: false`:** PavoPDF needs to read and write arbitrary PDF files that the user picks via the file dialog. The sandboxed security-scoped bookmark approach adds significant complexity. Post-launch, sandboxing + scoped bookmarks can be implemented to enable Mac App Store distribution; for the initial GitHub release it is not required.

**Why the hardened runtime keys:** macOS notarization requires hardened runtime. These three keys are the minimum set Tauri-based apps need to pass notarization without whitelisting specific entitlements.

## Chunk 2: Auto-Updater

### Task 4: Tauri Updater — Rust Feature Flag

**Files:**
- Modify: `src-tauri/Cargo.toml`
- Modify: `src-tauri/src/main.rs`

The auto-updater is **disabled by default** and can be excluded entirely from the binary with a Cargo feature flag. This means users who build from source never get update checks unless they opt in.

- [ ] **Step 1: Add the updater feature to `Cargo.toml`**

In `src-tauri/Cargo.toml`, ensure the `[features]` section includes:

```toml
[features]
default = []
# Enable to compile the auto-updater into the binary.
# Without this feature, the updater menu item and IPC commands are compiled out.
updater = ["tauri/updater"]

[dependencies]
tauri = { version = "2", features = ["tray-icon"] }
# ... rest of existing dependencies
```

- [ ] **Step 2: Gate the updater plugin registration behind the feature flag**

In `src-tauri/src/main.rs`, wrap the updater plugin with `#[cfg(feature = "updater")]`:

```rust
fn main() {
    tauri::Builder::default()
        .plugin(tauri_plugin_fs::init())
        .plugin(tauri_plugin_dialog::init())
        .plugin(tauri_plugin_shell::init())
        // Only compiled when `--features updater` is passed to cargo
        #[cfg(feature = "updater")]
        .plugin(tauri_plugin_updater::Builder::new().build())
        .invoke_handler(tauri::generate_handler![
            // ... existing commands
            #[cfg(feature = "updater")]
            commands::updater::check_for_update,
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

- [ ] **Step 3: Add the updater command module**

Create `src-tauri/src/commands/updater.rs`:

```rust
#[cfg(feature = "updater")]
use tauri_plugin_updater::UpdaterExt;

/// Called from the Settings UI to manually check for an update.
/// Returns `None` when already on latest, or the new version string.
#[cfg(feature = "updater")]
#[tauri::command]
pub async fn check_for_update(app: tauri::AppHandle) -> Result<Option<String>, String> {
    let updater = app.updater().map_err(|e| e.to_string())?;
    match updater.check().await {
        Ok(Some(update)) => Ok(Some(update.version.to_string())),
        Ok(None) => Ok(None),
        Err(e) => Err(e.to_string()),
    }
}

/// Performs the download-and-install sequence.
#[cfg(feature = "updater")]
#[tauri::command]
pub async fn install_update(app: tauri::AppHandle) -> Result<(), String> {
    let updater = app.updater().map_err(|e| e.to_string())?;
    if let Ok(Some(update)) = updater.check().await {
        update
            .download_and_install(|_chunk, _total| {}, || {})
            .await
            .map_err(|e| e.to_string())?;
    }
    Ok(())
}
```

Add `pub mod updater;` to `src-tauri/src/commands/mod.rs`.

- [ ] **Step 4: Add the updater Tauri plugin dependency**

In `src-tauri/Cargo.toml`:

```toml
[dependencies]
tauri-plugin-updater = { version = "2", optional = true }
```

And tie it to the feature:

```toml
[features]
updater = ["tauri/updater", "dep:tauri-plugin-updater"]
```


### Task 5: Updater Settings UI (Svelte)

**Files:**
- Modify: `src/lib/components/settings/UpdaterSettings.svelte`

- [ ] **Step 1: Write the UpdaterSettings component**

Create `src/lib/components/settings/UpdaterSettings.svelte`:

```svelte
<script lang="ts">
  import { invoke } from '@tauri-apps/api/core';
  import { settings, updateSettings } from '$lib/stores/settings';

  let checking = $state(false);
  let updateAvailable = $state<string | null>(null);
  let error = $state<string | null>(null);
  let installing = $state(false);

  // Whether the updater feature was compiled in
  // This will fail gracefully if the command doesn't exist
  let updaterAvailable = $state(true);

  async function checkNow() {
    checking = true;
    error = null;
    updateAvailable = null;
    try {
      const result = await invoke<string | null>('check_for_update');
      updateAvailable = result;
    } catch (e) {
      // If the command doesn't exist, the updater feature wasn't compiled in
      if (String(e).includes('not found')) {
        updaterAvailable = false;
      } else {
        error = String(e);
      }
    } finally {
      checking = false;
    }
  }

  async function installNow() {
    installing = true;
    error = null;
    try {
      await invoke('install_update');
      // App will restart automatically after install
    } catch (e) {
      error = String(e);
      installing = false;
    }
  }

  function toggleAutoUpdate(event: Event) {
    const enabled = (event.target as HTMLInputElement).checked;
    updateSettings({ autoUpdateEnabled: enabled });
  }
</script>

<section class="space-y-4">
  <h2 class="text-lg font-semibold text-gray-800">Updates</h2>

  {#if !updaterAvailable}
    <p class="text-sm text-gray-500">
      This build was compiled without the auto-updater. To enable updates, build with
      <code class="bg-gray-100 px-1 rounded text-xs">--features updater</code>.
    </p>
  {:else}
    <!-- Auto-update toggle -->
    <label class="flex items-center gap-3 cursor-pointer">
      <div class="relative">
        <input
          type="checkbox"
          class="sr-only peer"
          checked={$settings.autoUpdateEnabled}
          onchange={toggleAutoUpdate}
        />
        <div class="w-10 h-6 bg-gray-200 rounded-full peer
                    peer-checked:bg-teal peer-focus:ring-2 peer-focus:ring-teal/30
                    transition-colors after:content-[''] after:absolute after:top-0.5
                    after:left-0.5 after:bg-white after:rounded-full after:h-5 after:w-5
                    after:transition-transform peer-checked:after:translate-x-4">
        </div>
      </div>
      <div>
        <span class="text-sm font-medium text-gray-700">Automatic update checks</span>
        <p class="text-xs text-gray-500">
          PavoPDF will check for new versions on launch (version number only, no file data sent).
        </p>
      </div>
    </label>

    <!-- Manual check -->
    <div class="flex items-center gap-3 pt-2">
      <button
        onclick={checkNow}
        disabled={checking || installing}
        class="px-4 py-2 text-sm bg-teal text-white rounded-lg hover:bg-teal-dark
               disabled:opacity-50 disabled:cursor-not-allowed transition-colors"
      >
        {#if checking}
          Checking…
        {:else}
          Check for updates now
        {/if}
      </button>

      {#if updateAvailable}
        <div class="flex items-center gap-2">
          <span class="text-sm text-green-700 font-medium">
            v{updateAvailable} available
          </span>
          <button
            onclick={installNow}
            disabled={installing}
            class="px-3 py-1.5 text-sm bg-peach text-white rounded-lg
                   hover:bg-peach-dark disabled:opacity-50 transition-colors"
          >
            {installing ? 'Installing…' : 'Install & restart'}
          </button>
        </div>
      {:else if updateAvailable === null && !checking}
        <span class="text-sm text-gray-500">You are up to date.</span>
      {/if}
    </div>

    {#if error}
      <p class="text-sm text-red-600">{error}</p>
    {/if}

    <p class="text-xs text-gray-400">
      Current version: {__APP_VERSION__}
    </p>
  {/if}
</section>
```

**Note:** `__APP_VERSION__` is injected by Vite at build time. Add the following to `vite.config.ts` under `define`:

```ts

  __APP_VERSION__: JSON.stringify(process.env.npm_package_version),
},
```

### Task 6: Release Workflow

**Files:**
- Create/Replace: `.github/workflows/release.yml`

- [ ] **Step 1: Write the full release workflow**

Create `.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  # ─────────────────────────────────────────────────────────────────────────
  # Build the Tauri app on all three platforms in parallel
  # ─────────────────────────────────────────────────────────────────────────
  build-tauri:
    strategy:
      fail-fast: false
      matrix:
        include:
          - platform: ubuntu-22.04
            args: '--target x86_64-unknown-linux-gnu'
            rust-target: x86_64-unknown-linux-gnu

          - platform: macos-latest
            # Universal binary: Intel + Apple Silicon
            args: '--target universal-apple-darwin'
            rust-target: aarch64-apple-darwin x86_64-apple-darwin

          - platform: windows-latest
            args: '--target x86_64-pc-windows-msvc'
            rust-target: x86_64-pc-windows-msvc

    runs-on: ${{ matrix.platform }}

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install Rust stable
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.rust-target }}

      - name: Rust cache
        uses: swatinem/rust-cache@v2
        with:
          workspaces: './src-tauri -> target'

      # ── Linux: system libraries required by WebKitGTK and pdfium ──────────
      - name: Install Linux system dependencies
        if: matrix.platform == 'ubuntu-22.04'
        run: |
          sudo apt-get update
          sudo apt-get install -y \
            libwebkit2gtk-4.1-dev \
            libappindicator3-dev \
            librsvg2-dev \
            patchelf \
            libssl-dev \
            pkg-config \
            libasound2-dev \
            libgtk-3-dev \
            libglib2.0-dev

      - name: Install frontend dependencies
        run: npm ci

      # ── macOS: notarization (optional — only runs when secrets exist) ──────
      # To enable: add APPLE_ID, APPLE_ID_PASSWORD, APPLE_TEAM_ID,
      # APPLE_CERTIFICATE (base64 p12), APPLE_CERTIFICATE_PASSWORD to secrets.
      - name: Import Apple certificate (macOS only)
        if: matrix.platform == 'macos-latest' && env.APPLE_CERTIFICATE != ''
        env:
          APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
          APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
          KEYCHAIN_PASSWORD: ${{ secrets.KEYCHAIN_PASSWORD }}
        run: |
          CERTIFICATE_PATH=$RUNNER_TEMP/build_certificate.p12
          KEYCHAIN_PATH=$RUNNER_TEMP/app-signing.keychain-db
          echo -n "$APPLE_CERTIFICATE" | base64 --decode -o $CERTIFICATE_PATH
          security create-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
          security set-keychain-settings -lut 21600 $KEYCHAIN_PATH
          security unlock-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
          security import $CERTIFICATE_PATH -P "$APPLE_CERTIFICATE_PASSWORD" \
            -A -t cert -f pkcs12 -k $KEYCHAIN_PATH
          security list-keychain -d user -s $KEYCHAIN_PATH

      # ── Build ─────────────────────────────────────────────────────────────
      - name: Build Tauri app
        uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          # macOS notarization (optional)
          APPLE_ID: ${{ secrets.APPLE_ID }}
          APPLE_ID_PASSWORD: ${{ secrets.APPLE_ID_PASSWORD }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
          # Tauri updater signing key
          TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
          TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
        with:
          tagName: ${{ github.ref_name }}
          releaseName: 'PavoPDF ${{ github.ref_name }}'
          releaseBody: |
            See [CHANGELOG.md](https://github.com/${{ github.repository }}/blob/main/CHANGELOG.md)
            for release notes.

            ### Downloads
            | Platform | File |
            |---|---|
            | Windows | `PavoPDF_x64_en-US.msi` or `PavoPDF_x64-setup.exe` |
            | macOS | `PavoPDF_universal.dmg` |
            | Linux (AppImage) | `PavoPDF_amd64.AppImage` |
            | Linux (deb) | `PavoPDF_amd64.deb` |

            **macOS unsigned build:** If macOS shows a security warning, right-click the
            app and choose Open, then click Open in the dialog.
          releaseDraft: true
          prerelease: false
          args: ${{ matrix.args }}

  # ─────────────────────────────────────────────────────────────────────────
  # Build the website and deploy to Cloudflare Pages
  # ─────────────────────────────────────────────────────────────────────────
  deploy-website:
    runs-on: ubuntu-22.04
    needs: build-tauri
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Install Node.js 20

      - name: Deploy to Cloudflare Pages

          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
```

```markdown
- **OCR** — add a searchable text layer to scanned PDFs using Tesseract 5 (English + user-installed language packs).
- **Protect / Unlock** — add 128-bit AES password protection or remove owner passwords.
- Three-state UI: Dashboard → Workspace → Focused mode.
- Recent files list stored locally (last 20 entries).
- Dark-mode toggle (follows system preference by default).
- Auto-updater: opt-in, disabled by default, compiled out unless `--features updater` is passed.
- Native file dialogs for open and save operations.
- Platform bundles: Windows NSIS installer, macOS DMG (universal binary), Linux AppImage + deb.

### Notes

- Windows SmartScreen: may warn on first run. Code signing is planned post-launch.
- Estimated bundle size: ~120 MB (pdfium ~60 MB, Tesseract ~35 MB, app ~25 MB).
```

## Chunk 6: Final Release

### Task 12: Pre-Release Checklist

- [ ] **Step 1: Verify the build works locally on all platforms**

On each platform (or use CI artifacts), run:

```bash
cargo tauri build
```

Verify the installer launches, the app opens, and all 6 PDF tools work end-to-end.

- [ ] **Step 2: Replace `your-org` placeholders**

Search all files for `your-org` and replace with your actual GitHub username/organization:

```bash
grep -rl "your-org" . --include="*.json" --include="*.md" --include="*.svelte" \
  --include="*.yml" --include="*.ts" --include="*.js" | xargs sed -i 's/your-org/YOURORG/g'
```

Also update the updater `pubkey` field in `tauri.conf.json` with the public key generated in Task 6 Step 2.

- [ ] **Step 3: Set GitHub repository secrets**

In your GitHub repository → Settings → Secrets and variables → Actions, add:

| Secret name | Value | Required |
|---|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | Output of `cargo tauri signer generate` | Yes (for updater) |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Password chosen during key generation | Yes if password set |
| `APPLE_ID` | Your Apple ID email | No (post-launch) |
| `APPLE_ID_PASSWORD` | App-specific password for Apple ID | No (post-launch) |
| `APPLE_TEAM_ID` | Apple Developer Team ID | No (post-launch) |
| `APPLE_CERTIFICATE` | Base64-encoded Developer ID .p12 certificate | No (post-launch) |
| `APPLE_CERTIFICATE_PASSWORD` | .p12 export password | No (post-launch) |
| `KEYCHAIN_PASSWORD` | Any strong random password | No (post-launch) |
| `CLOUDFLARE_API_TOKEN` | Cloudflare Pages deploy token | Yes (for website) |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare account ID | Yes (for website) |

- [ ] **Step 4: Verify CI dry-run**

Push the branch to GitHub without a tag to confirm the workflow syntax is valid. The `release.yml` workflow only triggers on `v*` tags, so a plain push will not run it. You can verify the workflow syntax via:

```bash
gh workflow view release.yml
```

- [ ] **Step 5: Verify website builds locally**

```bash
cd website
npm install
npm run build
npm run preview
```

Open `http://localhost:4173` and verify all sections render correctly.


The workflow will:
1. Build on `ubuntu-22.04`, `macos-latest`, `windows-latest` in parallel (~15–25 min).
2. Upload installers to a **draft** GitHub Release named `PavoPDF v0.1.0`.
3. Deploy the website to Cloudflare Pages.

```bash
gh release view v0.1.0
```

Open the GitHub Releases page, review the draft, edit the release notes if needed, and click **Publish release**.

```bash
cargo tauri signer sign -k ~/.tauri/pavopdf.key path/to/PavoPDF_0.1.0_x64-setup.exe.tar.gz
```

Tauri's `tauri-action` does this automatically when `TAURI_SIGNING_PRIVATE_KEY` is set. Verify the `latest.json` file was uploaded to the release:

```bash
gh release view v0.1.0 --json assets --jq '.assets[].name'

```

- The website at pavopdf.com loads and all download links resolve.
